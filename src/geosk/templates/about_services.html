Web Map Service (WMS) is a technical specification defined by the Open Geospatial Consortium (OGC), which dynamically produces spatially referenced data maps from geographic information. This international standard defines a "map" as a representation of geographical information by returning a digital image suitable for display on a web browser.

Generally a WMS is not directly invoked; client applications such as GIS-Desktop or WEB-GIS are used that provide the user with interactive controls.

The Open Geospatial Consortium (OGC) Web Coverage Service (WCS) standard defines a standard interface and operations that allow interoperable access to geographical coverages. The term "grid coverage" typically refers to content such as satellite images, digital aerial photos, digital elevation data, and other phenomena represented by values at each measurement point.

Generally a WCS is not directly invoked; client applications such as GIS-Desktop or WEB-GIS are used that provide the user with interactive controls.

For Sensor Observation Service (SOS) we mean a technical specification defined by the OGC, applicable in the case where data are collected by sensors and they must be managed interoperable logic. This standard defines a web service that makes it possible to query the observations (using the Observation & Measurement - O&M scheme), the sensor metadata (using the Sensor Model Language - SensorML scheme) as well as the characteristics observed; still it allows to record a sensor, but also to cancel the recording, also defining the operations to insert new observations.

Generally an SOS to all requests responds with an XML, but for a more suitable display we recommend the use of client applications such as GIS-Desktop or WEB-GIS that provide the user with interactive controls.
